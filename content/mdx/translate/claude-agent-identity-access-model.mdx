- Claude Tag는 공유 채널에서 AI 에이전트가 팀과 협업할 때 개인 계정 대신 '에이전트 ID'라는 새로운 접근 모델을 도입합니다.
- 에이전트 ID는 AI가 개별 사용자 계정이 아닌 워크스페이스에 연결된 자체 계정을 통해 도구 및 컨텍스트에 접근하도록 합니다.
- 관리자는 워크스페이스 수준에서 에이전트 ID를 정의하고 채널별로 이를 재정의하여 코드 저장소 접근, API 키, 스킬 및 플러그인, 기본 설정 등을 세밀하게 제어할 수 있습니다.
- 이 모델은 AI의 자율성을 높이고 협업 팀 환경에 적합하며, 보안 및 감사 기록을 강화하여 누가 무엇을 했는지 명확하게 추적할 수 있습니다.
- DM(다이렉트 메시지)은 여전히 개인 claude.ai 계정을 사용하며, 에이전트 ID는 미래에 Just-in-Time credential grant와 같은 추가 보안 기능으로 확장될 예정입니다.
