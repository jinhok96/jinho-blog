Claude는 사용자의 프롬프트에 따라 다양한 방식으로 작동하는 '에이전트 루프'를 제공합니다.

- **턴 기반 루프**: 사용자가 각 단계를 직접 지시하며, Claude가 작업을 수행하고 사용자는 검증 후 다음 프롬프트를 입력합니다. SKILL.md를 활용하여 Claude가 자체 검증을 더 잘 수행하도록 할 수 있습니다.
- **목표 기반 루프 (/goal)**: 명확한 완료 기준과 최대 시도 횟수를 설정하여 복잡한 작업을 반복적으로 수행하도록 합니다. 완료 기준이 명확하면 Claude가 조기에 루프를 종료하는 것을 방지할 수 있습니다.
- **시간 기반 루프 (/loop, /schedule)**: 지정된 시간 간격으로 트리거되며, 주기적인 작업이나 외부 시스템 연동에 적합합니다. `/loop`은 로컬에서 실행되고 `/schedule`은 클라우드에서 실행됩니다.
- **능동형 루프**: 이벤트나 스케줄에 의해 실시간으로 인간의 개입 없이 트리거되며, 장기적으로 실행되는 반복 작업에 활용됩니다. `/schedule`, `/goal`, SKILL.md, 동적 워크플로우, 자동 모드를 조합하여 사용합니다.

코드 품질을 유지하기 위해서는 클린한 코드베이스 유지, SKILL.md를 통한 자체 검증 구현, 최신 문서 활용, 별도의 에이전트를 활용한 코드 리뷰가 중요합니다. 토큰 사용량 관리를 위해서는 적절한 루프와 모델 선택, 명확한 성공/종료 기준 설정, 파일럿 실행, 스크립트 활용, 불필요한 루프 실행 방지, 사용량 검토가 필요합니다.
