> 원문: [Denial of Service and Source Code Exposure in React Server Components](https://react.dev/blog/2025/12/11/denial-of-service-and-source-code-exposure-in-react-server-components) — React Blog

보안 연구원들이 지난주 치명적인 취약점 패치를 악용하려 시도하는 과정에서 React Server Components의 추가적인 두 가지 취약점을 발견하고 공개했습니다.

**이 새로운 취약점들은 원격 코드 실행을 허용하지 않습니다.** React2Shell에 대한 패치는 원격 코드 실행 취약점을 완화하는 데 효과적입니다.

새로운 취약점은 다음과 같이 공개되었습니다.

*   **서비스 거부 공격 (Denial of Service) - 높음**: [CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184), [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779), [CVE-2026-23864](https://www.cve.org/CVERecord?id=CVE-2026-23864) (CVSS 7.5)
*   **소스 코드 노출 (Source Code Exposure) - 중간**: [CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183) (CVSS 5.3)

새롭게 공개된 취약점의 심각성을 고려하여 즉시 업그레이드할 것을 권장합니다.

## 참고

### 이전에 발행된 패치는 취약합니다.

이전에 취약점을 해결하기 위해 업데이트했다면, 다시 한번 업데이트해야 합니다.

19.0.3, 19.1.4, 19.2.3 버전으로 업데이트한 경우, **이것들은 불완전하므로** 다시 한번 업데이트해야 합니다.

업그레이드 단계는 [이전 게시물](https://react.dev/blog/2025/12/03/critical-security-vulnerability-in-react-server-components#update-instructions)의 지침을 참조하십시오.

이 취약점들은 [CVE-2025-55182](https://react.dev/blog/2025/12/03/critical-security-vulnerability-in-react-server-components)와 동일한 패키지 및 버전에서 발견되었습니다.

여기에는 다음의 19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3 버전이 포함됩니다.

*   [react-server-dom-webpack](https://www.npmjs.com/package/react-server-dom-webpack)
*   [react-server-dom-parcel](https://www.npmjs.com/package/react-server-dom-parcel)
*   [react-server-dom-turbopack](https://www.npmjs.com/package/react-server-dom-turbopack?activeTab=readme)

수정 사항은 19.0.4, 19.1.5, 19.2.4 버전으로 백포트되었습니다. 위에 나열된 패키지를 사용하는 경우 즉시 수정된 버전으로 업그레이드하십시오.

이전과 마찬가지로, 앱의 React 코드가 서버를 사용하지 않는다면 이 취약점의 영향을 받지 않습니다. 앱이 React Server Components를 지원하는 프레임워크, 번들러 또는 번들러 플러그인을 사용하지 않는다면 이 취약점의 영향을 받지 않습니다.

## 참고

### 치명적인 CVE는 후속 취약점을 발견하는 것이 일반적입니다.

치명적인 취약점이 공개될 때, 연구원들은 초기 완화 조치가 우회될 수 있는지 테스트하기 위해 인접한 코드 경로를 면밀히 조사하여 변형된 공격 기법을 찾습니다.

이 패턴은 JavaScript뿐만 아니라 업계 전반에 걸쳐 나타납니다. 예를 들어, [Log4Shell](https://nvd.nist.gov/vuln/detail/cve-2021-44228) 이후, 커뮤니티가 원래의 수정을 테스트하면서 추가적인 CVE([1](https://nvd.nist.gov/vuln/detail/cve-2021-45046), [2](https://nvd.nist.gov/vuln/detail/cve-2021-45105))가 보고되었습니다.

추가 공개는 좌절감을 줄 수 있지만, 일반적으로 건강한 대응 주기의 신호입니다.

## 영향받는 프레임워크 및 번들러

일부 React 프레임워크 및 번들러는 취약한 React 패키지에 의존하거나, 피어 종속성을 가지거나, 포함하고 있었습니다. 다음 React 프레임워크 및 번들러가 영향을 받습니다: [next](https://www.npmjs.com/package/next), [react-router](https://www.npmjs.com/package/react-router), [waku](https://www.npmjs.com/package/waku), [@parcel/rsc](https://www.npmjs.com/package/@parcel/rsc), [@vite/rsc-plugin](https://www.npmjs.com/package/@vitejs/plugin-rsc), [rwsdk](https://www.npmjs.com/package/rwsdk).

업그레이드 단계는 [이전 게시물](https://react.dev/blog/2025/12/03/critical-security-vulnerability-in-react-server-components#update-instructions)의 지침을 참조하십시오.

## 호스팅 제공업체 완화 조치

이전과 마찬가지로, 여러 호스팅 제공업체와 협력하여 임시 완화 조치를 적용했습니다.

귀하는 앱을 보호하기 위해 이러한 조치에 의존해서는 안 되며, 즉시 업데이트해야 합니다.

## React Native

모노레포 또는 `react-dom`을 사용하지 않는 React Native 사용자의 경우, `package.json`에 `react` 버전이 고정되어 있어야 하며 추가적인 단계는 필요하지 않습니다.

모노레포에서 React Native를 사용하는 경우, 설치된 경우에만 영향을 받는 패키지를 업데이트해야 합니다.

*   `react-server-dom-webpack`
*   `react-server-dom-parcel`
*   `react-server-dom-turbopack`

이는 보안 권고 사항을 완화하기 위해 필요하지만, `react` 및 `react-dom`을 업데이트할 필요는 없으므로 React Native에서 버전 불일치 오류가 발생하지 않습니다.

자세한 내용은 [이슈](https://github.com/facebook/react-native/issues/54772#issuecomment-3617929832)를 참조하십시오.

# 높음: 다중 서비스 거부 공격

**CVE:** [CVE-2026-23864](https://www.cve.org/CVERecord?id=CVE-2026-23864) **기본 점수:** 7.5 (높음) **날짜**: 2026년 1월 26일

보안 연구원들은 React Server Components에 서비스 거부 취약점이 여전히 존재함을 발견했습니다.

이 취약점은 특별히 조작된 HTTP 요청을 Server Function 엔드포인트로 전송함으로써 트리거될 수 있으며, 취약한 코드 경로, 애플리케이션 구성 및 애플리케이션 코드에 따라 서버 충돌, 메모리 부족 예외 또는 과도한 CPU 사용을 초래할 수 있습니다.

2026년 1월 26일에 발행된 패치는 이러한 서비스 거부 취약점을 완화합니다.

## 참고

### 추가 수정 사항 발행

[CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184)의 서비스 거부 공격을 해결하는 초기 수정 사항은 불완전했습니다.

이로 인해 이전 버전들이 취약한 상태로 남아 있었습니다. 19.0.4, 19.1.5, 19.2.4 버전은 안전합니다.

# 높음: 서비스 거부 공격

**CVE:** [CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184) 및 [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779) **기본 점수:** 7.5 (높음)

보안 연구원들은 악의적인 HTTP 요청을 어떤 Server Functions 엔드포인트로든 전송할 수 있으며, React에 의해 역직렬화될 때 서버 프로세스를 중단시키고 CPU를 소비하는 무한 루프를 유발할 수 있음을 발견했습니다. 앱이 React Server Function 엔드포인트를 구현하지 않더라도, 앱이 React Server Components를 지원하는 경우 취약할 수 있습니다.

이는 공격자가 사용자에게 제품 액세스를 거부하고 잠재적으로 서버 환경에 성능 영향을 미칠 수 있는 취약점 벡터를 생성합니다.

오늘 발행된 패치는 무한 루프를 방지하여 이를 완화합니다.

# 중간: 소스 코드 노출

**CVE:** [CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183) **기본 점수:** 5.3 (중간)

한 보안 연구원은 취약한 Server Function으로 전송된 악의적인 HTTP 요청이 어떤 Server Function의 소스 코드를 안전하지 않게 반환할 수 있음을 발견했습니다. 악용하려면 Server Function이 명시적으로 또는 암묵적으로 문자열화된 인수를 노출하는 Server Function이 존재해야 합니다.

```
'use server';export async function serverFunction(name) {const conn = db.createConnection('SECRET KEY');const user = await conn.createUser(name); // 암묵적으로 문자열화되어 dbleak에 노출됨return {id: user.id,message: `Hello, ${name}!` // 명시적으로 문자열화되어 reply에 노출됨}}
```

공격자는 다음을 유출할 수 있습니다.

```
0:{"a":"$@1","f":"","b":"Wy43RxUKdxmr5iuBzJ1pN"}1:{"id":"tva1sfodwq","message":"Hello, async function(a){console.log(\"serverFunction\");let b=i.createConnection(\"SECRET KEY\");return{id:(await b.createUser(a)).id,message:`Hello, ${a}!`}}!"}
```

오늘 발행된 패치는 Server Function 소스 코드의 문자열화 방지를 통해 이를 해결합니다.

## 참고

### 소스 코드 내의 비밀 정보만 노출될 수 있습니다.

소스 코드에 하드코딩된 비밀 정보는 노출될 수 있지만, `process.env.SECRET`과 같은 런타임 비밀 정보는 영향을 받지 않습니다.

노출되는 코드의 범위는 Server Function 내부 코드로 제한되며, 번들러가 제공하는 인라이닝 양에 따라 다른 함수를 포함할 수 있습니다.

항상 프로덕션 번들을 확인하십시오.
