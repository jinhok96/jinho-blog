> 원문: [Tailwind CSS v4.3: Scrollbars, new colors, and more](https://tailwindcss.com/blog/tailwindcss-v4-3) — Tailwind CSS Blog

Tailwind CSS v4.2가 출시되었으며, v4.2 릴리스에 대한 블로그 포스팅을 잊었던 관계로 이번 포스트는 두 가지 릴리스에 걸친 새로운 기능들을 다룹니다.

v4.2에 포함된 내용은 다음과 같습니다:

*   [새로운 `mauve`, `olive`, `mist`, `taupe` 팔레트](#new-mauve-olive-mist-and-taupe-palettes) — gray, zinc, neutral, stone, slate가 만족스럽지 못할 때 사용할 수 있는 네 가지 뉴트럴 계열 팔레트입니다.
*   [최초의 webpack 플러그인](#first-class-webpack-plugin) — Next.js와 같은 프레임워크에서 성능을 크게 향상시킵니다.
*   [더 많은 논리적 속성 유틸리티](#more-logical-property-utilities) — `pbs-*`, `mbs-*`, `inline-*`, `block-*` 및 새로운 논리적 inset 유틸리티를 포함합니다.
*   [새로운 `font-features-*` 유틸리티](#new-font-features-utility) — 사용자 정의 CSS 작성 없이 저수준 OpenType 기능을 제어합니다.

… 그리고 v4.3의 새로운 내용은 다음과 같습니다:

*   [새로운 스크롤바 유틸리티](#new-scrollbar-utilities) — 어떤 브라우저 API를 사용해야 할지 고민할 필요 없이 스크롤바를 스타일링합니다.
*   [새로운 `@container-size` 유틸리티](#new-container-size-utility) — 컨테이너 쿼리가 높이까지 고려해야 할 때 크기 컨테이너를 생성합니다.
*   [새로운 `zoom-*` 유틸리티](#new-zoom-utilities) — 모든 브라우저가 존재에 동의했으므로 CSS `zoom` 속성을 마크업에서 직접 사용합니다.
*   [새로운 `tab-*` 유틸리티](#new-tab-utilities) — 탭 문자의 렌더링 너비를 제어합니다. GitHub처럼 8로 만들지는 말아주세요.
*   [Stacked + compound `@variant` 지원](#stacked-compound-variant-support) — CSS에서 stacked 및 compound variant를 직접 사용합니다.
*   [기능 유틸리티의 기본값](#default-values-for-functional-utilities) — 값과 함께 또는 값 없이 작동하는 유틸리티를 정의합니다.

Netflix와 Vercel 팀과의 협력을 통해 [Partners Program](https://tailwindcss.com/sponsor#partners)을 통해 v4.2 개선 사항의 많은 부분, 새로운 논리적 속성 유틸리티 및 `font-features-*` 지원 등이 이루어졌습니다. Tailwind CSS가 팀에 더 잘 맞도록 만드는 프레임워크 개선 사항이 있다면 기꺼이 도와드리겠습니다.

npm에서 최신 버전의 `tailwindcss`를 설치하여 프로젝트를 업그레이드하세요:

Tailwind CLI 사용

```
npm install tailwindcss@latest @tailwindcss/cli@latest
```

Vite 사용

```
npm install tailwindcss@latest @tailwindcss/vite@latest
```

PostCSS 사용

```
npm install tailwindcss@latest @tailwindcss/postcss@latest
```

webpack 사용


# [새로운 `mauve`, `olive`, `mist`, `taupe` 팔레트](#new-mauve-olive-mist-and-taupe-palettes)

v4.2에서는 기본 테마에 네 가지 새로운 색상 팔레트 `mauve`, `olive`, `mist`, `taupe`를 추가했습니다.

이들은 처음에 Tailwind Plus용으로 12월에 출시한 멀티 테마 마케팅 사이트 키트인 [Oatmeal](https://tailwindcss.com/plus/changelog#2025-12-18)에서 나왔으며, 기존 회색과 구별되면서도 뉴트럴에 가까운 팔레트가 필요했습니다.

이들은 모두 회색처럼 작동하지만 개성이 좀 더 있는, 유용한 뉴트럴 인접 범주에 속합니다:

50

100

200

300

400

500

600

700

800

900

950

mauve

olive

mist

taupe

전체 디자인을 좀 더 따뜻하거나, 시원하거나, 초록색이거나… mauver하게 만들고 싶을 때 기존 회색 팔레트를 대체하기 좋습니다.

```
<div class="bg-mauve-950 text-mauve-100 ...">Mauve</div><div class="bg-olive-100 text-olive-950 ...">Olive</div><div class="border border-mist-200 shadow-taupe-950/10 ...">Mist and taupe</div>
```

사용 가능한 모든 색상의 격자 형식은 [colors documentation](https://tailwindcss.com/docs/colors)에서 확인할 수 있습니다.

# [최초의 webpack 플러그인](#first-class-webpack-plugin)

Tailwind CSS v4.2는 webpack 프로젝트에 Tailwind CSS를 통합하기 위한 전용 `@tailwindcss/webpack` 로더를 추가했습니다.

이전에는 webpack 프로젝트에서 일반적으로 `postcss-loader`와 `@tailwindcss/postcss`를 통해 Tailwind을 실행했습니다. 이는 작동하지만 CSS가 PostCSS를 거치는 추가 단계를 필요로 하여 Tailwind이 이미 컴파일할 수 있는 문자열을 가져와야 했습니다.

전용 로더는 이러한 추가 작업을 건너뛰며, 복잡한 webpack 프로젝트에서 Tailwind 속도를 2배 이상 향상시키는 것을 확인했습니다. 예를 들어, Next.js와 Turbopack을 사용하여 tailwindcss.com 문서를 테스트한 결과는 다음과 같습니다:

Package

Build time

@tailwindcss/postcss

932ms

@tailwindcss/webpack

429ms

2.17x faster

사용 방법은 다른 webpack 로더와 동일한 설정입니다:

webpack.config.js

```
const MiniCssExtractPlugin = require("mini-css-extract-plugin");module.exports = {  plugins: [new MiniCssExtractPlugin()],  module: {    rules: [      {        test: /\.css$/i,        use: [MiniCssExtractPlugin.loader, "css-loader", "@tailwindcss/webpack"],      },    ],  },};
```

Turbopack이 webpack 로더를 호환성 계층을 통해 지원하므로 이러한 개선 사항은 Turbopack이 기본이 되고 있는 Next.js와 같은 프레임워크에도 적용됩니다.

더 자세한 내용은 [`@tailwindcss/webpack` README](https://github.com/tailwindlabs/tailwindcss/tree/main/packages/%40tailwindcss-webpack)를 확인하세요.

# [더 많은 논리적 속성 유틸리티](#more-logical-property-utilities)

Tailwind CSS v4.2는 다양한 쓰기 모드 및 방향에서 올바르게 적용되는 레이아웃을 더 쉽게 구축할 수 있도록 많은 새로운 논리적 속성 유틸리티를 추가했습니다.

간격을 위해 패딩, 마진, 스크롤 패딩, 스크롤 마진 및 테두리에 대한 새로운 block-start 및 block-end 유틸리티가 있습니다:

```
<div class="mbs-6 mbe-2 pbs-4 pbe-8 ...">  <!-- ... --></div>
```

```
<div class="scroll-mbs-4 scroll-mbe-4 scroll-pbs-12 scroll-pbe-12 ...">  <!-- ... --></div>
```

```
<div class="border-bs border-be-2 ...">  <!-- ... --></div>
```

또한 `inline-size` 및 `block-size`에 대한 논리적 크기 유틸리티와 min 및 max 변형을 추가했습니다:

```
<div class="block-64 inline-full max-block-screen max-inline-lg min-block-24 min-inline-0 ...">  <!-- ... --></div>
```

그리고 요소를 배치하기 위한 논리적 inset 유틸리티를 추가했습니다:

```
<div class="absolute inset-s-0 inset-e-4 inset-bs-2 inset-be-8 ...">  <!-- ... --></div>
```

기존 `start-*` 및 `end-*` 유틸리티는 여전히 사용할 수 있지만, 전체 API가 `inset-bs-*` 및 `inset-be-*`와 일치하도록 `inset-s-*` 및 `inset-e-*`로 대체되었습니다.

더 자세한 내용은 [`padding`](https://tailwindcss.com/docs/padding#using-logical-properties), [`margin`](https://tailwindcss.com/docs/margin#using-logical-properties), [`scroll-padding`](https://tailwindcss.com/docs/scroll-padding#using-logical-properties), [`scroll-margin`](https://tailwindcss.com/docs/scroll-margin#using-logical-properties), [`border-width`](https://tailwindcss.com/docs/border-width#using-logical-properties), [`inline-size`](https://tailwindcss.com/docs/inline-size), [`block-size`](https://tailwindcss.com/docs/block-size), [`top / right / bottom / left`](https://tailwindcss.com/docs/top-right-bottom-left#using-logical-properties) 문서를 참조하세요.

# [새로운 `font-features-*` 유틸리티](#new-font-features-utility)

`font-feature-settings`를 제어하기 위한 `font-features-*` 유틸리티도 추가했습니다:

normal

100.00

218.40

864.12

font-features-\["tnum"\]

100.00

218.40

864.12

```
<div class='font-features-["tnum"] ...'>  <!-- ... --></div>
```

표준 숫자와 같은 일반적인 경우에는 "tnum"이 있다는 것을 기억할 필요가 없으므로 `tabular-nums`와 같은 더 높은 수준의 유틸리티를 먼저 사용하는 것이 좋습니다.

하지만 폰트별 스타일 세트나 전용 유틸리티가 없는 다른 OpenType 기능을 활성화해야 하는 경우 `font-features-*`는 깔끔한 해결책을 제공합니다.

더 자세한 내용은 [`font-feature-settings`](https://tailwindcss.com/docs/font-feature-settings) 문서를 확인하세요.

* * *

Tailwind CSS v4.3은 CSS 스크롤바 API에 대한 기본 유틸리티를 추가하여, 두 번째 탭을 열고 어떤 브라우저가 무엇을 지원하는지 기억할 필요 없이 지루하지만 유용한 스크롤바 작업을 할 수 있게 되었습니다.

`scrollbar-width`를 제어하기 위해 `scrollbar-auto`, `scrollbar-thin`, `scrollbar-none`을 사용합니다:

Scroll vertically

The Cerulean Archives occupy three narrow floors above the old observatory, each lined with drawers of star maps, expedition notes, and brass instruments cataloged by hand.

On winter mornings, the staff rolls ladders between the shelves while pale light cuts through the roof windows and settles on the reading tables.

Visitors can request anything from the collection, but most come for the atlases that chart coastlines no longer found on modern maps.

Every returned volume is inspected, dusted, and wrapped before being carried back into the stacks for the next researcher.

```
<div class="scrollbar-thin scrollbar-thumb-sky-700 scrollbar-track-sky-100 overflow-auto ...">  <!-- ... --></div>
```

`scrollbar-thumb-*` 및 `scrollbar-track-*` 유틸리티를 사용하여 스크롤바 색상도 제어할 수 있습니다:

```
<div class="scrollbar-thumb-sky-700 scrollbar-track-sky-100 overflow-auto ...">  <!-- ... --></div>
```

일반적인 색상 불투명도 수정자도 함께 작동합니다:

```
<div class="scrollbar-thumb-slate-900/60 scrollbar-track-slate-900/10 ...">  <!-- ... --></div>
```

스크롤바가 나타날 때 레이아웃 이동을 방지하기 위해 `scrollbar-gutter-auto`, `scrollbar-gutter-stable`, `scrollbar-gutter-both`를 추가했습니다:

scrollbar-gutter-auto

Hey everyone! It's almost 2027 and we still don't know if there are aliens living among us, or do we? Maybe the person writing this is an alien. You will never know.

scrollbar-gutter-stable

```
<div class="scrollbar-gutter-stable overflow-auto ...">  <!-- ... --></div>
```

더 자세한 내용은 [`scrollbar-width`](https://tailwindcss.com/docs/scrollbar-width), [`scrollbar-color`](https://tailwindcss.com/docs/scrollbar-color), [`scrollbar-gutter`](https://tailwindcss.com/docs/scrollbar-gutter) 문서를 참조하세요.

# [새로운 `@container-size` 유틸리티](#new-container-size-utility)

Tailwind CSS v4.0에서는 `@container` 유틸리티로 기본 컨테이너 쿼리 지원을 추가했으며, 이는 인라인 크기 컨테이너를 생성합니다.

대부분의 경우 이것이 원하는 것이지만, `cqb` 및 `cqh`와 같은 컨테이너 쿼리 길이 단위는 컨테이너의 블록 크기에 따라 달라지며, 인라인 크기 컨테이너는 이 정보를 노출하지 않습니다.

따라서 v4.3에서는 크기 컨테이너를 생성하는 `@container-size`를 추가했습니다:

```
<div class="@container-size">  <div class="h-[50cqb]">    <!-- ... -->  </div></div>
```

`@container/{name}`과 마찬가지로 `@container-size/{name}`을 사용하여 크기 컨테이너의 이름을 지정할 수도 있습니다.

더 자세한 내용은 [responsive design docs](https://tailwindcss.com/docs/responsive-design#using-size-containers)를 참조하세요.

# [새로운 `zoom-*` 유틸리티](#new-zoom-utilities)

Tailwind CSS v4.3은 CSS `zoom` 속성에 대한 `zoom-*` 유틸리티를 추가했습니다. 이 속성은 2008년 Chrome 1에서 지원되었지만, 2024년에야 모든 주요 브라우저에서 지원이 완료된 오래된 브라우저 기능입니다.

```
<div class="zoom-75 ...">Zoomed out</div><div class="zoom-100 ...">Normal</div><div class="zoom-125 ...">Zoomed in</div>
```

임의 값 및 CSS 변수도 사용할 수 있습니다:

```
<div class="zoom-[1.1] ...">Zoomed in a little</div><div class="zoom-(--preview-zoom) ...">Zoomed using a variable</div>
```

더 자세한 내용은 [`zoom`](https://tailwindcss.com/docs/zoom) 문서를 참조하세요.

# [새로운 `tab-*` 유틸리티](#new-tab-utilities)

`tab-size`를 사용하여 탭 문자의 렌더링 너비를 제어하기 위한 `tab-*` 유틸리티도 추가했습니다.

이는 주로 코드 예제, 에디터 또는 실제 탭 문자가 포함될 수 있는 서식이 지정된 텍스트를 렌더링하는 모든 UI에 유용합니다:

tab-2

```
function indent() {	return 'tabbed';}
```

tab-8

```
function indent() {	return 'tabbed';}
```

```
<pre class="tab-2 ...">function indent() {&#10;&#9;return 'tabbed'&#10;}</pre><pre class="tab-8 ...">function indent() {&#10;&#9;return 'tabbed'&#10;}</pre>
```

임의 값 및 CSS 변수도 여기서 작동합니다:

```
<pre class="tab-[12px] ...">...</pre><pre class="tab-(--tab-size) ...">...</pre>
```

더 자세한 내용은 [`tab-size`](https://tailwindcss.com/docs/tab-size) 문서를 참조하세요.

# [Stacked + compound `@variant` 지원](#stacked-compound-variant-support)

Tailwind CSS v4.3은 마크업에서 직접이 아닌 CSS에서 variant를 사용할 때 `@variant`의 유연성을 높였습니다.

이제 stacked variant를 사용할 수 있습니다:

CSS

```
.button {  background: var(--color-sky-500);  @variant hover:focus {    background: var(--color-sky-600);  }}
```

Compiled CSS

```
.button {  background: var(--color-sky-500);  &:hover {    @media (hover: hover) {      &:focus {        background: var(--color-sky-600);      }    }  }}
```

또한 compound variant를 사용하여 여러 variant에 걸쳐 동일한 CSS 블록을 공유할 수 있습니다:

CSS

```
.button {  background: var(--color-sky-500);  @variant hover, focus {    background: var(--color-sky-600);  }}
```

Compiled CSS

```
.button {  background: var(--color-sky-500);  &:hover {    @media (hover: hover) {      background: var(--color-sky-600);    }  }  &:focus {    background: var(--color-sky-600);  }}
```

실제 컴포넌트를 만들고 Tailwind CSS 클래스로 직접 스타일링하는 것이 더 나은 경우가 많지만, CSS가 올바른 도구일 때는 `@variant`를 훨씬 더 편리하게 사용할 수 있습니다.

더 자세한 내용은 [`@variant`](https://tailwindcss.com/docs/adding-custom-styles#adding-custom-variants) 문서를 참조하세요.

# [기능 유틸리티의 기본값](#default-values-for-functional-utilities)

Tailwind CSS v4.0은 기능 `@utility` 정의를 도입했으며, 여기서 `--value(…)` 및 `--modifier(…)`를 사용하여 테마 값, 단순 값, 임의 값 및 수정자를 허용하는 유틸리티를 정의할 수 있습니다.

v4.3에서는 유틸리티가 값과 함께 또는 값 없이 작동해야 할 때 `--value(…)` 및 `--modifier(…)`에 `--default(…)`를 전달할 수 있습니다:

CSS

```
@utility tab-* {  tab-size: --value(integer, --default(4));}
```

이제 `tab` 유틸리티는 기본값으로 `4`를 사용하고, `tab-2`는 여전히 `2`로 해석됩니다:

HTML

```
<pre class="tab ...">...</pre><pre class="tab-2 ...">...</pre>
```

Generated CSS

```
.tab {  tab-size: 4;}.tab-2 {  tab-size: 2;}
```

이를 통해 기본 유틸리티가 명확한 작업을 수행하면서도 동일한 유틸리티 제품군에 대한 명시적 값을 지원하는 API를 더 쉽게 정의할 수 있습니다.

작은 변경 사항이지만 플러그인 작성자 및 사용자 정의 유틸리티를 작성하는 사람이 프레임워크 자체에서 제공하는 API와 훨씬 더 유사하게 느껴지는 API를 만들 수 있도록 합니다.

npm을 사용하여 최신 버전으로 업데이트하고 오늘부터 사용해보세요:

Tailwind CLI 사용

```
npm install tailwindcss@latest @tailwindcss/cli@latest
```

Vite 사용

```
npm install tailwindcss@latest @tailwindcss/vite@latest
```

PostCSS 사용

```
npm install tailwindcss@latest @tailwindcss/postcss@latest
```

Using webpack

```
npm install tailwindcss@latest @tailwindcss/webpack@latest
```

새로운 기능을 사용해보고 문제가 발생하는 부분을 알려주세요. 탭 크기가 4보다 크면 범죄이며 소프트웨어 엔지니어링 라이선스가 정지될 수 있다는 점을 기억하세요.
