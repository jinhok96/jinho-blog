> 원문: [Vanilla JavaScript support for Tailwind Plus](https://tailwindcss.com/blog/vanilla-js-support-for-tailwind-plus) — Tailwind CSS Blog

Tailwind Plus의 많은 UI 블록은 유용해지려면 JavaScript가 필요한데, 예를 들어 다이얼로그, 드롭다운, 커맨드 팔레트 등이 있습니다. React나 Vue 사용자가 아니라면 이러한 UI 블록을 사용하려면 복잡한 JavaScript를 직접 작성해야 했습니다.

이제 모든 것이 달라졌습니다. **Tailwind Plus의 모든 UI 블록이 이제 일반 HTML 예제를 포함하여 완벽하게 작동하고, 접근 가능하며, 상호작용 가능**합니다.

이제 [드롭다운](https://tailwindcss.com/plus/ui-blocks/application-ui/elements/dropdowns), [커맨드 팔레트](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/command-palettes), [다이얼로그](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/modal-dialogs), [드로어](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/drawers) 등을 JavaScript 프레임워크 없이도 프로젝트에 사용할 수 있습니다.

* * *

# [프레임워크 불필요](#no-framework-required)

이를 위해 저희는 [Tailwind Plus](https://tailwindcss.com/plus) 고객만을 위해 출시하는 라이브러리인 `@tailwindplus/elements`를 구축했습니다.

Elements는 HTML만으로 사용자 정의 가능한 상호작용 UI를 구축하는 데 필요한 복잡한 동작을 모두 캡슐화하는 헤드리스 [커스텀 엘리먼트](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements) 모음이며, 유틸리티 클래스나 사용자 정의 CSS를 사용하여 원하는 방식으로 스타일링할 수 있습니다.

이러한 커스텀 엘리먼트는 특정 JavaScript 프레임워크에 종속되지 않고 `<script>` 태그를 사용할 수 있는 모든 곳에서 작동합니다.

index.html

```
<script src="https://cdn.jsdelivr.net/npm/@tailwindplus/elements@1" type="module"></script>
```

Elements의 첫 번째 릴리스에서는 Tailwind Plus의 모든 UI 블록을 지원하기 위해 다음 기본 요소들을 제공합니다.

*   [Autocomplete](https://tailwindcss.com/plus/ui-blocks/documentation/autocomplete) — 커스텀 [Combobox](https://tailwindcss.com/plus/ui-blocks/application-ui/forms/comboboxes)와 같은 것을 만드는 데 사용됩니다.
*   [Command palette](https://tailwindcss.com/plus/ui-blocks/documentation/command-palette) — 커스텀 [Command Palettes](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/command-palettes)를 만드는 데 사용됩니다.
*   [Dialog](https://tailwindcss.com/plus/ui-blocks/documentation/dialog) — 커스텀 [Modal Dialogs](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/modal-dialogs), [Drawers](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/drawers) 등을 만드는 데 사용됩니다.
*   [Disclosure](https://tailwindcss.com/plus/ui-blocks/documentation/disclosure) — 접을 수 있는 [FAQ](https://tailwindcss.com/plus/ui-blocks/marketing/sections/faq-sections#component-8699d80b13ef524eb573e54b4d4b89d1), [Navbar](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/navbars)의 모바일 메뉴 등을 만드는 데 사용됩니다.
*   [Dropdown menu](https://tailwindcss.com/ui-blocks/documentation/dropdown-menu) — 물론 커스텀 [Dropdowns](https://tailwindcss.com/plus/ui-blocks/application-ui/elements/dropdowns)를 만드는 데 사용됩니다.
*   [Popover](https://tailwindcss.com/plus/ui-blocks/documentation/popover) — 커스텀 [Flyout Menus](https://tailwindcss.com/plus/ui-blocks/marketing/elements/flyout-menus) 등을 만드는 데 사용됩니다.
*   [Select](https://tailwindcss.com/plus/ui-blocks/documentation/select) — 커스텀 [Select Menus](https://tailwindcss.com/plus/ui-blocks/application-ui/forms/select-menus#component-8298198b136afab3bb19391ae716077f)를 만드는 데 사용됩니다.
*   [Tabs](https://tailwindcss.com/plus/ui-blocks/documentation/tabs) — 커스텀 텍스트 영역의 탭, [제품 개요](https://tailwindcss.com/plus/ui-blocks/ecommerce/components/product-overviews#component-13a89c2dc50a31afd66541dc28fa3c13) 등에서 사용하는 커스텀 탭을 만드는 데 사용됩니다.

Tailwind Plus 고객이라면 새롭게 추가된 [Elements 문서](https://tailwindcss.com/plus/ui-blocks/documentation/elements)에서 자세한 작동 방식과 예제를 확인해 보세요.

# [현대 웹 활용](#leveraging-the-modern-web)

Elements를 가능한 한 가볍고 네이티브하게 유지하기 위해 현대 플랫폼 기능을 많이 활용했습니다.

*   [커스텀 엘리먼트](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements)는 플랫폼 간 컴포넌트 추상화를 제공합니다.
*   [ `popover` 속성](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Global_attributes/popover)은 오버레이 및 팝오버를 자동으로 최상위 렌더링과 함께 관리하며, [`beforetoggle`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/beforetoggle_event) 이벤트로 전환을 제어합니다.
*   [기본 `<dialog>` 엘리먼트](https://developer.mozilla.org/en-US/docs/Web/API/HTMLDialogElement)는 포커스 트래핑 및 최상위 렌더링을 처리합니다.
*   [Invoker Commands](https://developer.mozilla.org/en-US/docs/Web/API/Invoker_Commands_API)는 사용자 정의 disclosure 토글과 같은 상호작용 엘리먼트를 선언적으로 관리합니다.
*   [`ElementInternals`](https://developer.mozilla.org/en-US/docs/Web/API/ElementInternals)는 사용자 정의 폼 컨트롤이 기본 폼 컨트롤처럼 작동하도록 합니다.

또한, 이러한 기능에 필요한 모든 폴리필을 번들링하여 Elements가 Tailwind CSS v4.0과 동일한 [브라우저](https://tailwindcss.com/docs/compatibility#browser-support)에서 작동하도록 보장합니다. 이는 이러한 현대 플랫폼 기능이 더 널리 사용됨에 따라 Elements의 크기가 계속 줄어들 것임을 의미합니다.

# [모든 곳에서 작동하는 컴포넌트](#components-that-work-everywhere)

HTML은 모든 웹 프레임워크 간의 가장 낮은 공통 분모이므로, Elements는 Tailwind Plus의 HTML 전용 UI 블록이 문자 그대로 어디에서든 작동하도록 만듭니다.
