> 이 글은 원문의 핵심 내용을 요약한 것입니다. 저작권은 원문 저자에게 있으며, 자세한 내용은 원문에서 확인하세요.

TanStack AI의 `useChat` hook이 업데이트되어 LLM의 구조화된 출력을 대화 기록에 보존합니다. 이전에는 새 메시지가 스트리밍될 때마다 이전 구조화된 출력이 사라져 개발자가 수동으로 상태를 관리해야 했습니다.

새로운 변경 사항은 다음과 같습니다.

*   **대화 기록 보존:** 이제 각 어시스턴트 턴의 구조화된 출력은 해당 `UIMessage`의 `parts` 배열에 자체적으로 보존됩니다. 이전 턴의 데이터가 자동으로 유지됩니다.
*   **향상된 타입 안정성:** 스키마 제네릭이 `messages[i].parts.find(p => p.type === 'structured-output').data`까지 확장되어 캐스팅 없이 타입이 안전한 데이터에 접근할 수 있습니다.
*   **간소화된 멀티턴 UI:** 개발자는 더 이상 별도의 상태 배열을 관리하거나 데이터 동기화를 신경 쓸 필요가 없습니다. `sendMessage()` 호출 시 이전 구조화된 응답이 LLM에 자동으로 전달되어 멀티턴 개선이 원활하게 이루어집니다.

이는 다음과 같은 시나리오에서 특히 유용합니다.

*   사용자가 대화를 통해 구조화된 객체를 반복적으로 수정하는 경우 (예: 레시피 생성, 디자인 사양 다듬기)
*   최신 결과뿐만 아니라 구조화된 객체의 전체 기록을 렌더링하려는 경우
*   LLM이 자신의 구조화된 응답을 기억하도록 하여 개발자의 수동 직렬화 부담을 줄이려는 경우

```typescript
// Client-side example snippet
const { messages } = useChat({
  outputSchema: RecipeSchema,
  // ... other options
});

// ... inside your component
messages.map((message) => {
  if (message.role === 'assistant') {
    const recipePart = message.parts.find(p => p.type === 'structured-output');
    if (recipePart?.data) {
      // Render the typed recipe data
      return <RecipeCard data={recipePart.data} />;
    }
  }
  // ... render other message types
});
```

이 업데이트는 React, Vue, Solid, Svelte에서 `useChat` 및 `createChat` hook을 통해 일관되게 적용됩니다. 자세한 내용은 원문에서 확인하세요.
