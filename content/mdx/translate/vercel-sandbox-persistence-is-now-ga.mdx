> 원문: [Sandbox persistence is now GA](https://vercel.com/changelog/sandbox-persistence-is-now-ga) — Vercel Blog

Vercel Sandbox는 이제 세션 간 파일시스템 상태를 자동으로 저장하고 복원합니다. 지속성은 기본적으로 활성화되어 있어 스냅샷을 관리하거나 상태를 수동으로 추적할 필요가 없습니다.

각 샌드박스는 프로젝트 내에서 고유한 참조 역할을 하는 내구성 있고 사용자 정의 가능한 이름을 가집니다. 이름을 통해 샌드박스를 생성, 검색 또는 재개할 수 있습니다. Vercel은 워크플로우를 방해하지 않고 자동으로 세션을 시작하고 중지합니다.

# 지속성 샌드박스 생성

`Sandbox.create()`를 호출하면 기본적으로 지속성이 활성화됩니다.

```
import { Sandbox } from "@vercel/sandbox";// 파일시스템이 자동으로 스냅샷됩니다const sandbox = await Sandbox.create({ name: "my-sandbox" }); await sandbox.runCommand("npm", ["install"]);await sandbox.stop();
```

기본적으로 지속성이 활성화된 샌드박스 생성

각 자동 스냅샷은 컴퓨팅과 별도로 청구되는 스냅샷 스토리지에 소모됩니다. 임시 워크로드의 경우 스토리지 비용을 최소화하기 위해 지속성을 선택 해제하십시오.

```
import { Sandbox } from "@vercel/sandbox";const sandbox = await Sandbox.create({ persistent: false });// 또는 기존 샌드박스를 업데이트합니다.await sandbox.update({ persistent: false });
```

임시 샌드박스 생성

CLI를 사용하여 지속성을 선택 해제하려면 `sandbox create`에 `--non-persistent`를 전달하십시오. 비지속성 샌드박스는 세션이 종료될 때 파일시스템을 삭제합니다.

# 지속성 샌드박스 재개

재개는 자동입니다. `runCommand()` 또는 `writeFiles()`와 같이 중지된 샌드박스에 대한 모든 호출은 가장 최근 스냅샷부터 새 세션을 시작합니다.

```
import { Sandbox } from "@vercel/sandbox";const resumedSandbox = await Sandbox.get({ name: "my-sandbox" });// 샌드박스를 자동으로 재개합니다.await resumedSandbox.runCommand("npm", ["test"]);
```

중지된 샌드박스를 자동으로 재개

# 기타 개선 사항

*   `Sandbox.fork()`: 기존 샌드박스에서 새 샌드박스 생성

*   `Sandbox.getOrCreate()`: 장기 실행 샌드박스를 위한 멱등성 검색 또는 생성

*   `Sandbox.delete()`: 샌드박스를 영구적으로 삭제

*   더 풍부해진 `sandbox.stop()`: 스냅샷 메타데이터와 활성 CPU 및 네트워크 전송 총계를 반환합니다.

*   수명 주기 훅: `create`, `get` 및 `getOrCreate`에 대한 `onCreate` 및 `onResume` 훅

*   [태그](https://vercel.com/docs/vercel-sandbox/concepts/tags): 멀티 테넌트 추적을 위해 샌드박스에 사용자 지정 속성 할당

# 시작하기

기본적으로 지속성 샌드박스를 생성하려면 최신 버전으로 업그레이드하십시오.

*   `pnpm install @vercel/sandbox@latest # SDK`

*   `pnpm install -g sandbox@latest # CLI`

[문서](https://vercel.com/docs/vercel-sandbox/concepts/persistent-sandboxes)에서 지속성 샌드박스에 대해 자세히 알아보십시오.
