# 2026년 Baseline과 접근성

웹을 구축한다는 것은 모두가 사용할 수 있는 경험을 구축하는 것을 의미합니다. [A11y Up의 최근 글](https://a11y-up.com/articles/baseline-and-accessibility-in-2026/)은 개발자가 웹 표준에 의존할 때 접근성 요구 사항을 고려하는 것이 더 효과적이라고 주장합니다. 얼마 동안 개발자들은 이제 웹 플랫폼의 일부가 된 접근 가능한 패턴을 다시 만들기 위해 사용자 정의되고 종종 무거운 JavaScript 솔루션을 배포해 왔습니다. 이러한 맞춤형 솔루션은 때때로 취약하고 보조 기술에서 깨지기 쉬우며 유지 관리하기 어렵습니다.

이 글은 웹 플랫폼 기능이 크로스 브라우저 상호 운용성을 달성함에 따라 접근성을 염두에 두고 개발하는 것이 더 효과적인 작업이 된다고 강조합니다. 공통 목표와 사용자 인터페이스 패턴을 달성하기 위해 웹 기능을 사용하면 많은 노력이 줄어들고 올바른 시맨틱스가 스크린 리더와 키보드 탐색 유틸리티에 직접 원활하게 노출됩니다. Baseline은 여기서 안내자 역할을 하며, 웹 기능이 프로젝트에서 평가하고 사용할 만큼 성숙한 시점을 알립니다.

# Baseline 신규 사용 가능 기능

이 섹션의 기능은 2026년 4월부터 핵심 브라우저 세트에서 지원되며 이제 Baseline 신규 사용 가능 기능입니다.

## CSS `contrast-color()` 함수

동적 테마 엔진과 사용자 정의 가능한 구성 요소는 높은 대비에 대한 사용자 선호도를 수용하기 위해 개발자가 여러 색상 시스템을 유지 관리하도록 강요했습니다. CSS `contrast-color()` 함수는 이러한 유지 관리 부담을 전적으로 브라우저로 이전합니다. 기본 입력 색상을 함수에 전달하면 엔진이 평가하여 높은 대비의 보조 색상을 반환하며, 일반적으로 가장 높은 가독성 점수를 제공하는 색상에 따라 검정 또는 흰색으로 매핑됩니다.

```
.card-header {
  background-color: var(--dynamic-bg-color);
  /* 자동으로 가장 높은 대비의 텍스트 색상으로 확인됩니다 */
  color: contrast-color(var(--dynamic-bg-color));
}

```

이를 통해 사용자 정의 솔루션이나 유지 관리하기 어려운 CSS 없이도 가독성을 위한 접근성 표준을 충족할 수 있습니다. 중간 톤 색상 선택에 계속 주의해야 하지만, 이 함수는 이러한 사용자 요구를 처리하는 데 필요한 보일러플레이트 CSS를 줄여줍니다. [MDN `contrast-color()` 참조 페이지](https://developer.mozilla.org/docs/Web/CSS/contrast-color)에서 더 자세히 알아볼 수 있습니다.

## `Math.sumPrecise()`

표준 루프 또는 `Array.prototype.reduce()`와 같은 메서드를 사용하여 숫자 시퀀스를 합산하면 부동 소수점 정밀도 손실이 발생할 수 있습니다. 이는 중요한 재무 계산 또는 원격 측정 합계에 영향을 미칠 수 있습니다.

`Math.sumPrecise()` 메서드는 이 문제를 해결합니다. 숫자의 반복 가능한 객체를 받아 정밀도 안전 루틴을 실행하여 정확한 합계를 제공합니다. [MDN `Math.sumPrecise()` 문서](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Math/sumPrecise)에서 메커니즘을 살펴보세요.

# Baseline 널리 사용 가능 기능

다음 기능은 Baseline 널리 사용 가능해져 프로젝트에서 광범위하게 호환되고 사용할 수 있게 되었습니다.

## `<search>` 요소

HTML `<search>` 요소는 웹 애플리케이션에서 검색 환경을 집합적으로 나타내는 폼 컨트롤, 필터링 메커니즘 및 제출 유틸리티에 대한 명시적 래퍼 역할을 합니다.

```
<search>
  <form action="/site-search">
    <label for="query">문서 검색</label>
    <input type="search" id="query" name="q">
    <button>검색</button>
  </form>
</search>

```

컨테이너 요소를 `<search>` 태그로 전환하면 사용자에게 접근성 이점을 제공합니다. 브라우저는 `search`의 암시적 ARIA 랜드마크 역할을 요소에 자동으로 할당하여 `<form>` 요소에 `role="search"`를 지정할 필요가 없습니다. 이를 통해 스크린 리더는 검색 인터페이스를 식별하고 사용자가 탐색하도록 도울 수 있습니다. [MDN `<search>` 요소 페이지](https://developer.mozilla.org/docs/Web/HTML/Element/search)에서 구현 세부 정보를 읽어보세요.

## 웹 인증 공개 키 액세스

`AuthenticatorAttestationResponse` 인터페이스에 대한 직접 속성 추출기의 광범위한 지원 덕분에 웹 인증(WebAuthn) API를 사용하여 비밀번호 없는 인증이 이제 덜 복잡해졌습니다. `getPublicKey()` 및 `getPublicKeyAlgorithm()`과 같은 메서드를 사용하면 브라우저가 원시 이진 데이터를 처리하지 않고도 공개 키 세부 정보를 추출합니다. [MDN `AuthenticatorAttestationResponse` 페이지](https://developer.mozilla.org/docs/Web/API/AuthenticatorAttestationResponse)에서 이러한 속성과 사용 방법에 대해 자세히 알아보세요.

## `String.prototype.isWellFormed()` 및 `String.prototype.toWellFormed()`

JavaScript 문자열은 UTF-16으로 인코딩되며, 이는 유니코드 쌍으로 복잡한 문자 및 이모티콘을 매핑합니다. 문자열이 이를 고려하지 않고 슬라이스되면 대리 쌍의 고립된 절반(고립된 대리자라고 함)이 남아 잘못 형성된 텍스트가 생성됩니다.

`isWellFormed()`를 사용하면 개발자가 문자열에 고립된 대리자가 포함되어 있는지 확인할 수 있으며 부울 값을 반환합니다. 문자열이 유효성 검사를 통과하지 못하면 `toWellFormed()`를 호출하여 잘못된 대리자를 표준 유니코드 대체 문자(`U+FFFD`)로 바꿀 수 있습니다. 이는 `encodeURI()`와 같은 함수를 호출하기 전에 유용하며, 잘못 형성된 입력을 발견하면 `URIError`를 발생시킵니다. [MDN `String.prototype.isWellFormed()` 문서](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/String/isWellFormed)에서 이러한 메서드가 작동하는 방식을 알아보세요.

## ARIA 속성 반영

대화형 요소의 접근성 상태를 업데이트하려면 표준 DOM 속성 메서드(예: `element.setAttribute('aria-expanded', 'true')`)를 통한 왕복이 필요했습니다. ARIA 속성 반영은 접근성 속성을 객체 속성으로 미러링하여 이를 단순화합니다.

`Element` 인터페이스는 ARIA 속성을 `element.ariaExpanded`, `element.ariaChecked`, `element.ariaHidden`과 같은 인스턴스 속성으로 직접 반영합니다. 이를 통해 점 표기법 구문을 사용하여 접근성 상태를 수정할 수 있습니다.

```
// 깔끔하고 읽기 쉬운 상태 업데이트
toggleButton.ariaExpanded = toggleButton.ariaExpanded === "true" ? "false" : "true";

```

ARIA 대상을 JavaScript 속성으로 처리하면 UI 프레임워크와 상태 관리 도구가 보조 컨텍스트를 더 안정적으로 조정할 수 있으며, 스크린 리더 컨텍스트를 실제 애플리케이션 상태와 정렬하는 데 도움이 됩니다. 반영된 속성의 전체 목록을 보려면 [MDN `Element` 인스턴스 속성 가이드](https://developer.mozilla.org/docs/Web/API/Element#instance_properties)를 확인하세요.
